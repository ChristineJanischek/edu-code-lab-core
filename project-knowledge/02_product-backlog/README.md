# Product Backlog Workflow

## Ordnerstruktur

1. `inbox/`: neue Ideen und Projektbausteine
2. `analyzed/`: analysierte und geschaetzte Bausteine
3. `planned/`: freigegebene Bausteine mit Umsetzungsfenster
4. `completed/`: umgesetzte Bausteine mit Lernergebnis

## Routine

1. Neue Idee immer zuerst als Projektbaustein in `inbox/` erfassen.
2. Woechentliche Analyse: Bedarf, Reichweite, Risiko, Abhaengigkeiten.
3. Priorisieren mit dem Scoring-Modell.
4. Bei Freigabe nach `planned/` verschieben.
5. Nach Umsetzung nach `completed/` verschieben und Lernergebnis dokumentieren.

## Priorisierungsmodell (0-3 pro Kriterium)

Kriterien:

1. Bedarf
2. Entlastung
3. Lernwirkung
4. Reichweite
5. Wiederverwendbarkeit
6. Machbarkeit
7. Risiko

Formel:

`Prioritaet = Bedarf + Entlastung + Lernwirkung + Reichweite + Wiederverwendbarkeit + Machbarkeit - Risiko`

## Mindestanforderung fuer Analyseabschluss

Ein Projektbaustein gilt als analysiert, wenn folgende Felder befuellt sind:

1. Problem und Beduerfnis
2. Zielgruppe
3. Systemebene
4. Akzeptanzkriterien
5. Risiken
6. Erprobungsplan
